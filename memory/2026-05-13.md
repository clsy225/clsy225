# 2026-05-13

- User reported a critical Raspberry Pi maintenance rule: do **not** run plain `sudo apt-get update && sudo apt-get upgrade` on this setup because it can break the system.
- Safe update workflow for this machine:
  1. `sudo apt-mark hold raspberrypi-bootloader`
  2. `sudo apt-get update`
  3. `sudo apt-get upgrade`
- Reason observed during package install attempts: apt fetches over IPv6/HTTP were failing, and user explicitly says the hold step is required on this Raspberry Pi setup to avoid damage.
- Reference added for display driver/docs: LCDWiki 3.5inch RPi Display (MPI3501) documentation: http://www.lcdwiki.com/zh/3.5inch_RPi_Display
- LCD details from doc: 3.5 inch 320x480 SPI display, ILI9486 + XPT2046, supports Raspberry Pi 5B, online/offline driver installation via `goodtft/LCD-show` / `LCD35-show`, and rotate via `rotate.sh`.
